# NOVA Language Specification

### Version 0.4

NOVA v0.4 expands upon v0.3 by introducing:

* Maps / Objects
* Optional Properties
* Nested Maps
* Maps Containing Arrays
* Property Access
* Property Mutation

---

# New Features

## Maps

Maps store structured key-value data.

---

## Basic Map

```text
user: M = {
    name: S = "Varun",
    age: N = 19,
    isCoder: B = true
}
```

Map definitions use `:`.

Map assignments use `=`.

---

## Optional Properties

Optional properties use `?`.

```text
user: M = {
    name: S,
    age: N,
    nickname?: S
}
```

Valid:

```text
user = {
    name = "Varun",
    age = 19
}
```

Also valid:

```text
user = {
    name = "Varun",
    age = 19,
    nickname = "Nova"
}
```

---

## Nested Maps

```text
user: M = {
    name: S,

    address: M = {
        city: S,
        country: S
    }
}
```

---

## Maps Containing Arrays

```text
user: M = {
    name: S = "Varun",

    skills: [S] = [
        "JavaScript",
        "TypeScript",
        "Python"
    ]
}
```

---

## Arrays Of Maps

```text
users: [M] = [
    {
        name = "Varun",
        age = 19
    },

    {
        name = "Nova",
        age = 21
    }
]
```

---

## Mixed Structures

```text
data: [U] = [
    {
        name = "Varun",
        skills = [
            "JS",
            "TS"
        ]
    },

    {
        name = "Nova",
        skills = [
            "Python"
        ]
    }
]
```

---

## Map Keys

Map keys must follow the same naming rules as variables.

Valid:

```text
user_name
user1
first_name
```

Invalid:

```text
1user
user-name
user@name
```

Rules:

* Keys may contain letters (`a-z`, `A-Z`)
* Keys may contain numbers (`0-9`)
* Keys may contain underscores (`_`)
* Keys may not start with a number
* Special characters are not allowed

---

## Property Access

Properties are accessed using dot notation.

```text
user.name

user.age
```

Nested access:

```text
user.address.city
```

Array access inside maps:

```text
user.skills[0]
```

Arrays of maps:

```text
users[0].name
```

---

## Property Mutation

Map properties may be modified if the map is mutable.

```text
user: M = {
    name: S = "Varun",
    age: N = 19
}

user.name = "Nova"
```

Result:

```text
{
    name = "Nova",
    age = 19
}
```

---

Datatype restrictions still apply.

```text
user.age = "twenty"
```

Error:

```text
Datatype mismatch.
```

---

## Immutable Maps

Maps declared as constants are immutable.

```text
user:: M = {
    name: S = "Varun",
    age: N = 19
}
```

Invalid:

```text
user.name = "Nova"
```

Error:

```text
Cannot modify immutable collection.
```

Deep immutability applies to all nested arrays and maps contained within a constant collection.
