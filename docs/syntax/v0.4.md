# NOVA Language Specification

### Version 0.4

NOVA v0.4 expands the language by introducing schema maps, map instances, optional properties, nested schemas, arrays of maps, property access, and property mutation.

Schemas provide NOVA's first structured datatype, allowing programs to model real-world objects while preserving runtime type safety.

---

# New Features

## Schema Maps

Schemas define the structure and datatypes of map instances.

Schemas are declared using the `M` datatype.

```text
User: M = {
    name: S,
    age: N,
    isCoder: B
}
```

Schemas define structure only and do not store values.

Schema names may be used anywhere a datatype is expected.

```text
user: User

users: [User]

groups: [[User]]
```

---

## Map Instances

Variables may use schema names as datatypes.

```text
User: M = {
    name: S,
    age: N,
    isCoder: B
}

user: User = {
    name = "Varun",
    age = 19,
    isCoder = true
}
```

Every required property must be present.

Every value must match the declared schema.

---

## Optional Properties

Optional properties are declared using `?`.

```text
User: M = {
    name: S,
    age: N,
    nickname?: S
}
```

Both declarations are valid.

```text
user: User = {
    name = "Varun",
    age = 19
}
```

```text
user: User = {
    name = "Varun",
    age = 19,
    nickname = "Nova"
}
```

---

## Nested Schemas

Schemas may reference other schemas.

```text
Address: M = {
    city: S,
    country: S
}

User: M = {
    name: S,
    address: Address
}
```

```text
user: User = {
    name = "Varun",

    address = {
        city = "Mumbai",
        country = "India"
    }
}
```

---

## Schemas Containing Arrays

Schemas may contain arrays.

```text
User: M = {
    name: S,
    skills: [S]
}
```

---

## Arrays of Maps

Arrays may contain schema instances.

```text
User: M = {
    name: S,
    age: N
}

users: [User] = [
    {
        name = "Varun",
        age = 19
    },

    {
        name = "Nova",
        age = 21
    }
]
```

Nested arrays of maps are also supported.

```text
groups: [[User]]
```

---

## Property Access

Properties are accessed using dot notation.

```text
user.name

user.address.city

users[0].name

user.skills[0]
```

Property access and array indexing may be freely combined.

---

## Property Mutation

Mutable map instances may be modified.

```text
user.name = "Nova"
```

Assigned values must match the property's datatype.

```text
user.age = "twenty"
```

Produces:

```text
Datatype mismatch.
```

---

## Property Validation

Map instances are validated against their schema.

Missing required properties produce an error.

```text
Missing required property 'age'.
```

Unknown properties produce an error.

```text
Unknown property 'city'.
```

---

## Immutable Maps

Maps declared as constants cannot be modified.

```text
user:: User = {
    name = "Varun",
    age = 19
}
```

```text
user.name = "Nova"
```

Produces:

```text
Cannot modify immutable collection.
```

Nested collections inside immutable maps are also immutable.
