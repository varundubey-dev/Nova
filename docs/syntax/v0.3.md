# NOVA Language Specification

### Version 0.3

NOVA v0.3 expands the language by introducing arrays, typed arrays, multi-type arrays, nested arrays, array indexing, and array mutation.

Arrays become NOVA's first collection type, enabling programs to store and manipulate ordered groups of values.

---

# New Features

## Arrays

Arrays store ordered collections of values.

Arrays may contain:

* Primitive values
* Arrays

---

## Untyped Arrays

`[U]` stores values of any datatype.

```text
items: [U] = [
    1,
    "hello",
    true,
    null
]
```

---

## Typed Arrays

Typed arrays restrict every element to a single datatype.

```text
numbers: [N] = [
    1,
    2,
    3
]

names: [S] = [
    "A",
    "B",
    "C"
]
```

---

## Empty Arrays

Arrays may be created without elements.

```text
numbers: [N] = []

names: [S] = []

items: [U] = []
```

---

## Multi-Type Arrays

Arrays may accept multiple datatypes.

```text
mixed: [N, S] = [
    1,
    "hello"
]
```

Elements outside the allowed datatypes produce an error.

```text
mixed: [N, S] = [
    true
]
```

Produces:

```text
Invalid datatype inside array.
```

---

## Nested Arrays

Arrays may contain other arrays.

```text
matrix: [[N]] = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]
```

---

## Array Indexing

Arrays use zero-based indexing.

```text
numbers[0]

numbers[1]
```

Nested indexing is also supported.

```text
matrix[0][1]
```

---

## Array Mutation

Mutable arrays may be modified.

```text
numbers: [N] = [
    1,
    2,
    3
]

numbers[0] = 10
```

Result:

```text
[10, 2, 3]
```

Assigned values must satisfy the array datatype.

```text
numbers[0] = "hello"
```

Produces:

```text
Datatype mismatch.
```

---

## Immutable Arrays

Arrays declared as constants cannot be modified.

```text
numbers:: [N] = [
    1,
    2,
    3
]
```

```text
numbers[0] = 10
```

Produces:

```text
Cannot modify immutable collection.
```
