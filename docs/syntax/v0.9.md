# NOVA Language Specification

### Version 0.9

NOVA v0.9 expands the language by introducing modules, imports, exports, and the first importable Standard Library modules.

---

# New Features

## Module Imports

Modules may be imported using the `import` keyword.

```text
import math
```

Imported modules are accessed using dot notation.

```text
math.sqrt(25)
```

---

## Selective Imports

Specific exported declarations may be imported directly from a module.

```text
import greet from examples.utils
```

Imported declarations become available in the current scope.

```text
greet()
```

Only exported declarations may be imported.

---

## Module Paths

Local modules are referenced using dotted module paths.

```text
import greet from examples.utils
```

```text
import factorial from examples.math
```

Module paths are resolved relative to the project root.

File extensions are omitted.

For example:

```text
examples.utils
```

resolves to:

```text
examples/utils.nova
```

---

## Module Aliases

Imported modules or declarations may be assigned an alias using the `as` keyword.

```text
import math as m
```

```text
m.sqrt(25)
```

Selective imports may also be aliased.

```text
import greet as hello from examples.utils
```

```text
hello()
```

---

## Exported Declarations

Declarations may be exported using the `export` keyword.

```text
export fn greet() {

    print("Hello")

}
```

Only exported declarations are visible outside the module.

Non-exported declarations remain private to the defining module.

---

## Exported Constants

Constants may be exported.

```text
export PI:: N = 3.1415926535
```

```text
import math

print(
    math.PI
)
```

---

## Exported Schemas

Schemas may also be exported.

```text
export User: M = {
    name: S,
    age: N
}
```

```text
import User from models.user
```

```text
person: User = {

    name = "Nova"
    age = 1

}
```

---

## Standard Library Modules

NOVA's Standard Library is now organized into importable modules.

```text
import math
```

```text
math.sqrt(81)
```

Standard Library modules follow the same import rules as user-defined modules.

---

## Module Resolution

Imported modules are resolved before program execution begins.

Imported declarations become available before runtime execution starts.

If a module cannot be located, compilation fails.

```text
import missing
```

Produces:

```text
Module 'missing' could not be found.
```

---

## Circular Import Detection

Circular module dependencies are detected before execution.

Invalid:

```text
a.nova
```

```text
import b
```

```text
b.nova
```

```text
import a
```

Produces:

```text
Circular module dependency detected.
```
