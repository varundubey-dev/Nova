# NOVA Language Specification

### Version 0.7

NOVA v0.7 expands upon v0.6 by introducing:

* Function Declarations
* Function Calls
* Parameters
* Arguments
* Return Values
* Local Function Scope
* Global Scope Access
* Recursion
* Early Returns

---

# New Features

## Function Declaration

Functions are declared using the `fn` keyword.

```text
fn greet() {
    print("Hello")
}
```

Functions are executed only when called.

---

## Function Call

```text
fn greet() {
    print("Hello")
}

greet()
```

Output:

```text
Hello
```

---

## Parameters

Functions may accept parameters.

```text
fn greet(name: S) {
    print("Hello", name)
}
```

Usage:

```text
greet("Varun")
```

Output:

```text
Hello Varun
```

---

## Multiple Parameters

```text
fn add(a: N, b: N) {
    print(a + b)
}
```

Usage:

```text
add(5, 10)
```

Output:

```text
15
```

---

## Return Values

Functions may return values using `return`.

```text
fn add(a: N, b: N) -> N {
    return a + b
}
```

Usage:

```text
result: N = add(5, 10)

print(result)
```

Output:

```text
15
```

---

## Return Enforcement

Functions declaring a return type must return a value.

Valid:

```text
fn add(a: N, b: N) -> N {
    return a + b
}
```

Invalid:

```text
fn add(a: N, b: N) -> N {
}
```

Error:

```text
Function 'add' expected return value of type N.
```

---

## Datatype Checking

Arguments must match parameter datatypes.

```text
fn greet(name: S) {
    print(name)
}
```

Invalid:

```text
greet(10)
```

Error:

```text
Datatype mismatch.
```

---

## Function Scope

Variables declared inside a function exist only inside that function.

```text
fn test() {
    temp: N = 10
}
```

Invalid:

```text
print(temp)
```

Error:

```text
Variable 'temp' is not declared.
```

---

## Accessing Global Variables

Functions may access global variables.

```text
name: S = "Varun"

fn greet() {
    print(name)
}
```

Output:

```text
Varun
```

---

## Variable Shadowing

Local variables may shadow global variables.

```text
name: S = "Global"

fn test() {
    name: S = "Local"

    print(name)
}
```

Output:

```text
Local
```

The global value remains unchanged.

---

## Functions Before Definition

Functions may be called before their definition.

```text
greet()

fn greet() {
    print("Hello")
}
```

Output:

```text
Hello
```

Function declarations are resolved before program execution begins.

---

## Recursive Functions

Functions may call themselves.

```text
fn countdown(n: N) {

    if n > 0 {

        print(n)

        countdown(n - 1)

    }

}
```

Usage:

```text
countdown(5)
```

Output:

```text
5
4
3
2
1
```

---

## Recursive Return Values

Recursive functions may return values.

```text
fn factorial(n: N) -> N {

    if n <= 1 {
        return 1
    }

    return n * factorial(n - 1)

}
```

Usage:

```text
print(
    factorial(5)
)
```

Output:

```text
120
```

---

## Returning Early

A function immediately exits when `return` is executed.

```text
fn test() -> N {

    return 10

    print("Never Runs")

}
```

Result:

```text
10
```

Statements appearing after `return` are not executed.

---

## Void Functions

Functions without `->` return no value.

```text
fn greet() {
    print("Hello")
}
```

Such functions are intended only for side effects.

---

## Function Arguments

Arguments are evaluated before being passed to a function.

```text
fn add(a: N, b: N) -> N {
    return a + b
}

print(
    add(
        5 + 5,
        10 + 10
    )
)
```

Output:

```text
30
```

---

## Function Calls Inside Expressions

Function calls may be used anywhere an expression is allowed.

```text
fn double(value: N) -> N {
    return value * 2
}

result: N = (
    double(10) +
    double(5)
)

print(result)
```

Output:

```text
30
```

---

## Nested Function Calls

Function calls may be nested.

```text
fn add(a: N, b: N) -> N {
    return a + b
}

print(
    add(
        add(1, 2),
        add(3, 4)
    )
)
```

Output:

```text
10
```

---

## Scope Resolution

Functions create their own runtime scope.

When a variable is accessed:

1. The current function scope is searched.
2. Parent scopes are searched.
3. The global scope is searched.

```text
name: S = "Global"

fn greet() {

    print(name)

}
```

Output:

```text
Global
```

---

## Runtime Environments

Each function call creates a new runtime environment.

```text
fn test() {

    value: N = 10

}
```

Variables declared inside a function are isolated from other function calls.

This behavior enables recursion and local state management.

---

## Recursion Requirements

Recursive functions must eventually reach a terminating condition.

Valid:

```text
fn countdown(n: N) {

    if n > 0 {

        countdown(n - 1)

    }

}
```

Invalid:

```text
fn infinite() {

    infinite()

}
```

The second example never terminates and will eventually exhaust the runtime call stack.

---

## Source-Aware Diagnostics

All function-related parser, type, and runtime errors provide precise source locations.

Example:

```text
fn greet(name: S) {
    print(name)
}

greet(10)
```

Output:

```text
Type Error: Datatype mismatch.

--> example.nova:5:7
5 | greet(10)
          ^
```
